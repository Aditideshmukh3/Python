* Variables rules :
1.	Variables name should be start with alphabets a-z or A-Z.
2.	Variables name should not start with digit (0-9).
3.	You cannot use any symbols in variables name except _ (preferrably in middle of two words).
4.	Variables name should not contain keywords ( there are 30+ keywords available like for, if, else, break, continue, etc.).
5.	Variables name / python is case sensitive (like a=10, A=10 is different for python).

* File handling :
A) Function file -
   1. read()
   2. write()
   3. open()
   4. close()
B) Modes - (used with open function)
   1. read -    'r' -  Opens file for reading (error if file doesn’t exist)
   2. write -   'w' -  Opens file for writing (creates new or erases old content)
   3. append -  'a' -  Opens file for adding content at the end (keeps old content)  
   4. create -  'x' -  Creates a new file (error if file already exists)  

   open('abc.txt', 'r')
   open('abc.txt', 'w')
   open('abc.txt', 'a')
   open('abc.txt', 'x')

* Oops (Object Oriented Programming Language) logic :
oops concept is build on - class & object
- security
- reusability
- Reliability

Four pillars of oops logic :
1. Inheritance - One class can use properties and methods of another class / inherit something
    parent class - child class / base class - derived class

    types of Inheritance :
    1. Single Inheritance
    2. multiple Inheritance
    3. Multilevel Inheritance
    4. Heirarchical Inheritance
    5. Hybrid Inheritance

2. Polymorphism - Same function name or method but behaves differently for different objects with different meanings
3. Abstraction - Hiding complex details and showing only the essential features / hides unnecessary data
    Two types :
    1. Data Abstraction - original data entity is hidden
    2. Process Abstraction - hides implementation

4. Encapsulation - Wrapping data and methods into one unit and protecting them from outside access / security to data __
